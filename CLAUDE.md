pyt# SMM694 RAG Prototype — Claude Code Guide

## Project overview
Retrieval-Augmented Generation (RAG) prototype built for SMM694 Applied NLP assignment.
Ingests PDF documents, embeds them into a ChromaDB vector store, retrieves relevant chunks at query time, and generates answers via an LLM. A Streamlit dashboard (`app.py`) exposes the pipeline interactively. RAGAS evaluation is in `evaluate.py`.

## Stack
- **Embeddings / vector store**: ChromaDB (`chroma_db/` — git-ignored, regenerated by `src/embed.py`)
- **LLM**: configured via `ANTHROPIC_API_KEY` in `.env`
- **UI**: Streamlit (`app.py`)
- **Evaluation**: RAGAS (`evaluate.py`)

## Key files
| File | Purpose |
|---|---|
| `src/download.py` | Download 10-K filings from SEC EDGAR → `data/raw/` |
| `src/ingest.py` | Parse raw filings → text chunks |
| `src/embed.py` | Embed chunks → ChromaDB |
| `src/retrieve.py` | Query vector store → top-k chunks |
| `src/generate.py` | Chunks + prompt → LLM answer |
| `src/utils.py` | Shared helpers (logging, config loading) |
| `app.py` | Streamlit entry point |
| `evaluate.py` | RAGAS evaluation script |

## Running the project
```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Copy and fill in API keys (ANTHROPIC_API_KEY, SEC_USER_NAME, SEC_USER_EMAIL)
cp .env.example .env

# 3. Download 10-K filings from SEC EDGAR
python src/download.py

# 4. Parse filings into chunks and embed into ChromaDB
python src/ingest.py
python src/embed.py

# 5. Launch the app
streamlit run app.py

# 6. Run evaluation
python evaluate.py
```

## Git-ignored paths
- `venv/`, `.env`, `__pycache__/`, `*.pyc`
- `data/raw/` — SEC EDGAR filings (HTML/txt, large files)
- `data/processed/` — chunked JSON (derived artifact)
- `chroma_db/` — vector store (derived artifact)
